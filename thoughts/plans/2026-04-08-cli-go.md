# English Punch CLI (Go)

## Overview

Go CLI (`ep`) for English Punch App. Convex backend is unchanged — CLI communicates via Convex HTTP API directly.

## Scope

- **In scope**: auth (login/logout/status), bag listing
- **Out of scope (TODO)**: card CRUD, learning/review, stats, MCP server, analytics, shell completions

---

## Architecture

Based on `~/works/cli-best-practices.md`. Layered: `cmd/` (UX) → `convex/` (HTTP) → `config/` (credentials).

```
cli/
├── cmd/ep/main.go              # Clean entry point: main() → run()
├── internal/ep/
│   ├── cmd/
│   │   ├── root.go             # Root command, global flags (--output, --config-dir)
│   │   ├── auth.go             # ep auth login / logout / status
│   │   └── bags.go             # ep bags list
│   ├── convex/
│   │   └── client.go           # Convex HTTP API client (action/query/mutation)
│   ├── config/
│   │   ├── config.go           # Viper config loading (convex_url, etc.)
│   │   └── keychain.go         # macOS Keychain read/write/delete
│   └── common/
│       ├── output.go           # Table/JSON output formatting
│       └── errors.go           # Semantic exit codes (ExitError)
├── go.mod
├── go.sum
├── Makefile                    # build, install, release targets
└── .goreleaser.yaml            # Cross-compile + Homebrew tap
```

## Auth Strategy

### Why Keychain + Re-auth Per Invocation

Convex `@convex-dev/auth` Password provider issues short-lived JWTs. Storing a token means frequent expiry and re-login. Instead:

1. `ep auth login` → prompt email/password → verify against Convex → store in macOS Keychain
2. Every command reads credentials from Keychain → calls Convex `signIn` action → gets fresh JWT → uses it for the request
3. `ep auth logout` → remove from Keychain

### Keychain Details

- Service name: `english-punch-cli`
- Account: user's email
- Password: user's password
- Access: `security` CLI (`security add-generic-password`, `find-generic-password`, `delete-generic-password`)
- Fallback: `~/.config/english-punch/credentials.json` with `0600` permissions (for Linux future support)

### Convex signIn Flow (HTTP)

MCP server의 인증 방식을 Go HTTP로 재현:

```
POST {CONVEX_URL}/api/action
Content-Type: application/json

{
  "path": "auth:signIn",
  "args": {
    "provider": "password",
    "params": { "email": "...", "password": "...", "flow": "signIn" }
  }
}

Response: { "value": { "tokens": { "token": "jwt..." } } }
```

Then for authenticated queries:

```
POST {CONVEX_URL}/api/query
Content-Type: application/json
Authorization: Bearer {jwt}

{
  "path": "auth:loggedInUser",
  "args": {}
}

Response: { "value": { "_id": "users:xxx", "email": "..." } }
```

## Commands

### `ep auth login`

```
$ ep auth login
Email: user@example.com
Password: ********
Authenticated as user@example.com
```

- Interactive prompt (detect TTY; error if not terminal with message to use `--email`/`--password` flags)
- Validate credentials against Convex before storing
- Store in Keychain on success

### `ep auth logout`

```
$ ep auth logout
Logged out.
```

- Remove credentials from Keychain

### `ep auth status`

```
$ ep auth status
Logged in as user@example.com
Convex URL: https://strong-otter-914.convex.cloud
```

- Read from Keychain, verify against Convex
- Exit code 0 if authenticated, 2 if not

### `ep bags list`

```
$ ep bags list
Name              Total  New  Learning  Review  Active
TOEFL Vocab         120   30        15      75  yes
Daily Phrases        45   10         5      30  yes

$ ep bags list -o json
[{"id":"bags:xxx","name":"TOEFL Vocab","totalCards":120,...}]
```

- Calls `learning:getUserBags` query
- Supports `--output table|json` (table is default)
- Requires auth (auto-reads from Keychain, signs in, queries)

## Convex HTTP Client

Thin wrapper around Convex HTTP API. No Go SDK exists — raw HTTP.

```go
type Client struct {
    BaseURL    string
    HTTPClient *http.Client
    Token      string // JWT, set after signIn
}

func (c *Client) Action(ctx context.Context, path string, args any) (json.RawMessage, error)
func (c *Client) Query(ctx context.Context, path string, args any) (json.RawMessage, error)
func (c *Client) Mutation(ctx context.Context, path string, args any) (json.RawMessage, error)

// Auth convenience
func (c *Client) SignIn(ctx context.Context, email, password string) error  // calls signIn action, sets c.Token
func (c *Client) GetCurrentUser(ctx context.Context) (*User, error)         // calls loggedInUser query
```

## Config

Viper config with layered precedence:

```
CLI flags  >  Env vars (EP_*)  >  Config file  >  Defaults
```

Config file: `~/.config/english-punch/config.yaml`

```yaml
convex_url: https://strong-otter-914.convex.cloud  # default
```

- `--config-dir` flag for alternate profiles
- `EP_CONVEX_URL` env var override

## Error Handling

```go
const (
    ExitSuccess        = 0
    ExitGeneralError   = 1
    ExitAuthError      = 2  // not logged in, invalid credentials
    ExitConnectionError = 3  // Convex unreachable
)
```

## Homebrew Distribution

### goreleaser

`.goreleaser.yaml`로 cross-compile + Homebrew formula 자동 생성:

```yaml
brews:
  - repository:
      owner: echoja
      name: homebrew-tap
    homepage: https://github.com/echoja/english-punch-app
    description: CLI for English Punch flashcard app
```

### Installation

```bash
brew tap echoja/tap
brew install english-punch
```

### Release Flow

1. Tag: `git tag cli/v0.1.0`
2. goreleaser builds darwin-arm64, darwin-amd64 (Linux later)
3. Uploads binaries to GitHub Release
4. Auto-updates Homebrew formula in `echoja/homebrew-tap`
